Что прописать в amoCRM

Воронка v6 → конкретная настройка amoCRM. Принцип: amoCRM хранит и показывает + простые авто-задачи; весь «ум» (раздача, рейтинг, таймеры, ИИ, бот) — наш движок на donula.online, связь через вебхуки и API.

1Воронка — этапы

Открытые этапы 1–7 + два служебных. Закрытие — с причиной.

1Новый лид 2В работе 3Принимает решение 4Договор на подписи 5Договор подписан — платёж сегодня 6Оплачено 990 — добор до 13 500 7Сделка закрыта → сопровождение
↩ Дозвонщики (бот-автодозвон) ♻︎ Общий пул (ждёт переназначения) ✕ Закрыто/не реализовано — причина: отказ · не наш · недозвон · расторжение

Статус 7 → передача в отдельную воронку «Сопровождение» (другой отдел). «Общий пул» — этап без ответственного, откуда движок раздаёт. «Закрыто» — нативное, с обязательной причиной (loss reason).

3Кто что делает

Главная развилка: что тянет нативная amoCRM, а что — наш движок.

amoCRM сама (Цифровая воронка + Salesbot)

  • Авто-задача при входе в статус (что делать на этапе)
  • Авто-сообщения, Salesbot: мгновенный ответ на входящее
  • Закрытие с причиной (loss reason)
  • Хранит поля, чек-листы, историю, записи звонков
  • Показывает менеджеру карточку + задачу

Движок donula.online (этого amoCRM НЕ умеет)

  • Умная раздача: кап 10/день + кап на руках + по рейтингу; пул тёплых по рейтингу
  • Таймеры исходов: забил 3 дня → пул · «Завис» N ч → пул · 14 дней → пул · недозвон 5/3 → закрытые
  • Детект «движения»: реальное касание (звонок+длительность из телефонии) vs попытка · документ · платёж
  • Рейтинг: ИИ-балл звонка + ведение + исходы + расторжения → тир
  • ИИ-разбор звонка (запись → рубрика → балл), генерация заключения
  • Бот-эскалация горячего + «горячий момент» (дата оплаты подошла → флаг + приоритет)

4Вебхуки и события (стык)

amoCRM → движокновый лид · смена статуса · новое входящее сообщение · результат задачи · оплата (если через amo)
Телефония → движокзвонок start/end + agent_id + запись (для ИИ-разбора и детекта реального касания)
Движок → amoCRM (API)назначить ответственного · сменить статус (в пул / Завис / закрытые) · поставить флаг «Горячий» · создать задачу «Сейчас делай» с текстом и скриптом · записать оценку звонка / рейтинг / тир
Бот ↔ клиентвходящее → ответ мгновенно · отправка заключения со слов + ссылки на оплату · инструкция по КИ · ловит «готов платить» → флаг + задача менеджеру
Граница одной фразой: amoCRM = воронка + поля + простые авто-задачи + Salesbot (хранит и показывает). Движок = раздача лидов, рейтинг, все «умные» таймеры, ИИ-разбор и заключение, бот-эскалация (весь ум). Стык — вебхуки amoCRM/телефонии в движок и обратная запись по API.
