Воронка и этапы в amoCRM

настройка
  • 7 этапов: Новый → В работе → Принимает решение → Договор на подписи → Платёж сегодня → Оплачено 990 (добор) → Сделка закрыта
  • + служебные: Дозвонщики (бот-автодозвон) и Общий пул (без ответственного, движок раздаёт)
  • Закрытие — с причиной: отказ · не наш · недозвон · расторжение
→ Спека amoCRM→ Воронка v6
2

Поля карточки

настройка
  • 14 полей: сумма долга · тариф · следующий шаг+дата · флаги «Горячий»/«Завис» · договор/декларация · оплаты+даты · остаток · КИ · документы · расторжение · попытки дозвона
  • Имя · Фамилия · Сумма долга — понадобятся для подстановки в клиентские страницы (шаги 4–5)
  • Рейтинг/тир/% расторжений — на менеджере, считает движок
→ Список полей
Движок — старт раздачи
3

Система распределения лидов

разработка
  • Вебхук новой заявки → движок назначает ответственного
  • Кап: 10 новых/день + не больше ~25–30 на руках
  • Старт — round-robin (без рейтинга, без наказаний). Рейтинг-наклон добавим на шаге 7
Решение перед стартом: тянет ли нативная раздача amoCRM «10/день + кап на руках», или сразу мини-сервис на donula.online (скорее — сервис).
Клиентские страницы — закрытие и догон
4

Презентация — с данными из полей

прототип готов
  • Менеджер открывает презентацию по ссылке — внутри подставляются {Имя} {Фамилия} {сумма долга} {тариф} из карточки
  • Поток закрытия: условия → договор → подпись по СМС (ОКИ ДОКИ) → оплата (СБП)
  • Это инструмент статуса 4 «Договор на подписи»
→ Презентация→ Оплата (СБП)
Доделать: подстановка полей из amoCRM (URL-параметры/API) + боевые ОКИ ДОКИ и СБП вместо демо.
5

Страницы фин-защиты — догон на оплату 990 / 13 500

прототипы готовы
  • Заключение со слов (бесплатно, статусы 3/5) и по кредитной истории (после 990, статус 6) — показывают клиенту его выгоду, шлёт бот
  • Портал «ваше дело» — ощущение, что им занялись, ведёт к доплате
  • Ссылки на оплату 990 / 13 500 прямо в заключении и боте
  • Везде подставляются {Имя} {Фамилия} {сумма долга} из полей
→ Портал клиента→ Заключение со слов→ Заключение по КИ
Доделать: подстановка полей + отправка ботом + боевая ИИ-генерация заключения.
ИИ, рейтинг, бот

Постобработка ИИ — разбор звонка

демо окна готово
  • Телефония (запись) → ИИ по рубрике → окно разбора менеджеру («82% по этапам») — коучинг
  • Сначала в shadow (считает, НЕ режет лиды) — калибровка ИИ↔РОП
→ Демо окна разбора
Сделать: реальная speech/ИИ-аналитика на ваших записях звонков.
7

Рейтинг + распределение по рейтингу

разработка
  • Агрегатор: ИИ-балл звонка + ведение + исходы + расторжения → тир
  • Тёплые / из пула / горячие лиды → лучшим закрывальщикам (наклон к раздаче из шага 3)
  • Пока без наказаний — позитивное распределение
→ Логика рейтинга в воронке
