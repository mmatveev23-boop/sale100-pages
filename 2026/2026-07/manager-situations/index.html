Не купил — переходим в дожим и обязательно запрашиваем КИ, подписал он или нет. Пришла КИ — та же ссылка показывает точные числа. 🔗Ссылка одна — на весь цикл клиента

Менеджер отправляет клиенту одну ссылку на первом звонке и больше её не пересобирает и не переотправляет. Она сама меняет вид по мере продвижения сделки — по статусу в CRM (подписан? оплачен?) и наличию КИ. Для дожима менеджер просто говорит: «откройте ту же ссылку».

Запросили КИ в дожиме, клиент прислал — та же ссылка показывает точные числа и кредиторов, ничего пересылать не надо. Ниже разные ссылки — только для демо (в проде это один короткий URL, состояние берётся из сделки).

Как это устроено

  • Два этапа. Первый звонок (ситуация 1) — целиком со слов, открывается с начала: эксперт ведёт клиента по всем экранам: выгода → договор → подпись → оплата (полная или частичка). Дожим (ситуации 2–5): не подписал (ситуация 2) — снова полный флоу, уже по КИ; подписал (ситуации 3–5) — открывается сразу на мини юр заключении: выгода + «договор подписан ✓» + кнопка оплаты, без повторной подписи.
  • КИ — обязательный шаг дожима. Охотники КИ не берут, первый звонок идёт со слов. Не купил на первом звонке — запрашиваем у клиента кредитную историю обязательно, даже без подписанного договора. Пришла — та же ссылка показывает точные числа, и дожим идёт по фактам.
  • Дожим-сообщения — в один клик из виджета. Три готовых текста с цифрами клиента: «не открыл» (ссылка + выгода), «не подписал» (рост долга + просьба заказать КИ с инструкцией + пример юр. заключения), «по КИ» (заключение готово + оплата). Кнопки ✉ в виджете копируют текст в буфер.
  • Одна ссылка, а не набор документов. Состояние = данные (есть ли КИ) × прогресс сделки (подписан? оплачен?). Клиенту всегда уходит одна и та же ссылка.
  • Ссылки собирает виджет автоматически. Со слов — по умолчанию; загрузил КИ — добавляется ?ki=1; статус подписи/оплаты — из сделки. Эта страница показывает, какая ссылка = какое состояние.
  • Параметры демо: ?ki=1 — по кредитной истории; ?signed=1 — подписал, не оплатил; ?paid=1 — оплачено; ?name&debt&pay — цифры клиента со слов. В проде их подставляет движок из сделки.
  • Продукт (РДГ / Списание) выбирается на диагностике; здесь обе ссылки даны, чтобы посмотреть оба.
