Как это устроено

  • Два этапа. Первый звонок (ситуации 1–3) открывается с начала — эксперт ведёт клиента по экранам: выгода → договор → подпись → оплата (полная или частичка). Дожим (ситуации 4–6) открывается сразу на мини-заключении: выгода сверху + «договор подписан ✓» + кнопка оплаты, без повторной подписи.
  • Одна ссылка, а не набор документов. Состояние = данные (есть ли КИ) × прогресс сделки (подписан? оплачен?). Клиенту всегда уходит одна и та же ссылка.
  • Ссылки собирает виджет автоматически. Со слов — по умолчанию; загрузил КИ — добавляется ?ki=1; статус подписи/оплаты — из сделки. Эта страница показывает, какая ссылка = какое состояние.
  • Параметры демо: ?ki=1 — по кредитной истории; ?signed=1 — подписал, не оплатил; ?paid=1 — оплачено; ?name&debt&pay — цифры клиента со слов. В проде их подставляет движок из сделки.
  • Продукт (РДГ / Списание) выбирается на диагностике; здесь обе ссылки даны, чтобы посмотреть оба.
